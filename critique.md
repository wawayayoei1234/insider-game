# คำวิจารณ์และข้อเสนอแนะระบบเกม Insider Party (จาก Antigravity AI)

บันทึกคำวิจารณ์ฉบับนี้จัดทำขึ้นเพื่อส่งต่อการวิเคราะห์และการทดสอบระบบให้กับทางทีมงาน หรือใช้ประกอบการพัฒนาต่อผ่านระบบ AI อื่น ๆ (เช่น Claude Code) ในเวิร์กสเปซนี้

---

## 1. ผลการทดสอบลูปการเล่นจริง (Gameplay Simulation Results)
เราได้ทำการจำลองสถานการณ์การเล่นของผู้เล่น 4 คน (Alice, Bob, Charlie, Dave) ผ่านการจำลองยิง WebSocket ไปยัง Server (Go backend) บนพอร์ต 3001
* **ผลลัพธ์:** ตัวเกมสามารถดำเนินสถานะและจัดการรอบการเล่นแบบครบวงจรได้อย่างถูกต้อง: `Lobby` -> `Assign Roles` -> `Countdown` (ทายคำ) -> `Voting` -> `Scoreboard` -> `Lobby`
* **การคิดคะแนน:** ฝั่ง Server คำนวณแจกแต้มของทั้งฝ่าย Commons และฝ่าย Insider ได้ถูกต้องตามกฎกติกาเมื่อการโหวตสิ้นสุด

---

## 2. จุดเด่น/ข้อดีของระบบปัจจุบัน (Strengths)
1. **Per-recipient snapshot redaction (ความปลอดภัยสูง):** เซิร์ฟเวอร์ Go กรองและปิดบังคำลับ (`SecretWord`) รวมถึงบทบาทของผู้เล่นอื่นก่อนส่งสัญญาณกระจายออกไป ทำให้ผู้เล่นฝ่าย Commons ไม่สามารถโกงเกมผ่านการแกะข้อมูล JSON ใน Chrome DevTools ได้
2. **ระบบกู้คืนที่นั่ง (Session Reconnection):** การใช้ localStorage และ Reconnect Token ทำให้ผู้เล่นที่เน็ตหลุดหรือกดรีเฟรชกลับมาเล่นต่อจากคะแนนเดิม บทบาทเดิม และไอดีเดิมได้ทันที
3. **ระบบยกเลิกรอบอัจฉริยะ (Active Round Voiding):** หากกรรมการ (Judge) หรือ Insider ตัดการเชื่อมต่อกลางรอบ ระบบจะยกเลิกการทายและส่งทุกคนกลับห้อง Lobby ทันที เพื่อป้องกันรอบค้าง

---

## 3. ข้อวิจารณ์และจุดที่ควรปรับปรุง (Critiques & Recommendations)

### 📱 3.1 การแสดงผลบนอุปกรณ์พกพา (Mobile Responsiveness)
* **ปัญหา:** เลย์เอาต์หลักของแอปถูกฟิกซ์เป็น 3 คอลัมน์ (Left Players: 92px/112px, Center, Right Players: 92px/112px) เมื่อรันบนหน้าจอขนาดเล็กแนวตั้ง (เช่น โทรศัพท์มือถือขนาดกว้าง ~360px-400px) คอลัมน์ฝั่งผู้เล่นทั้งสองข้างจะเบียดพื้นที่ตรงกลาง (ซึ่งแสดงแผงควบคุมเกม แชท และฟังก์ชันการโหวต) จนเหลือสัดส่วนที่แคบมากและใช้งานได้ลำบาก
* **ข้อเสนอแนะ:** เมื่อเปิดใช้งานผ่านหน้าจอมือถือ (`xs` screen) ควรซ่อนคอลัมน์ผู้เล่นสองฝั่ง แล้วปรับให้แสดงเป็น Grid หรือแถบสไลด์แนวนอน (Horizontal Scroll) ด้านบนสุดแทน เพื่อให้ส่วนคอนเทนต์และแชทตรงกลางใช้งานได้อย่างเต็มตา

### 💬 3.2 ปุ่ม Lobby View ถูก Modal Dialog บดบังช่องทางสื่อสาร
* **ปัญหา:** ในสถานะก่อนเริ่มเกม หน้าจอปรับแต่งสถานะห้อง (`LobbyView.js`) ถูกแสดงผลผ่าน MUI `<Dialog open>` ซึ่งมี Backdrop สีดำทึบล็อกหน้าจอเบื้องหลังไว้ ส่งผลให้ผู้เล่นในห้องไม่สามารถใช้ช่องทางแชทพิมพ์พูดคุยหรือส่งความรู้สึก (Reactions) กันได้ขณะรอเริ่มรอบ
* **ข้อเสนอแนะ:** ควรเรนเดอร์องค์ประกอบการตั้งค่าของ Lobby ลงในหน้ากระดานส่วนกลางโดยตรง (Inline content) แทนการครอบด้วย Modal Dialog เพื่อรักษาประสบการณ์การสื่อสารระหว่างผู้เล่นให้ราบรื่น

### ⚙️ 3.3 บั๊กแฝง: โหมดโหวตค้างคอยเวลาในกรณีเดดล็อก (Deadlock Voting Delay)
* **ปัญหา:** ในโค้ด `main.go` หากเกิดกรณีที่จำนวนผู้เล่นที่มีสิทธิ์โหวตลดลงเหลือ 0 (เช่น ทุกคนคะแนนเท่ากันหมดจากการเสมอกันและถูกบล็อกจากการโหวตก่อนหน้า: `eligibleVoters == 0`) เงื่อนไขในการนับคะแนนอัตโนมัติจากการส่งโหวตจะไม่ทำงาน (เนื่องจากติดเงื่อนไขตรวจสอบ `eligible > 0` ในส่วนรับข้อความ `vote_insider`) ทำให้ผู้เล่นทั้งหมดในห้องต้องนั่งเวลารอคอยไปจนกว่าจะหมดเวลา 90 วินาที เพื่อให้ Timer ของห้องบนเซิร์ฟเวอร์รันหมดลงเองจึงจะเปลี่ยนสถานะเกมได้
* **ข้อเสนอแนะ:** ปรับปรุงตรรกะใน `main.go` หากประเมินค่า `eligibleVoters(room) == 0` ตั้งแต่ตอนเริ่มโหวตหรือเสร็จสิ้นลูปการโหวตรอบล่าสุด ให้ข้ามขั้นตอนจับเวลาแล้วสั่งคำนวณสุ่มผลคะแนนทันที

### 🏆 3.4 คะแนนสะสมของคนเดาคำปริศนาถูก (Scoring Discrepancy)
* **ปัญหา:** ใน `plan.md` ระบุตกลงไว้ว่า *"Judge และคนทายถูก (correctGuesserId) จะไม่ได้แต้มเลย"* แต่ในตรรกะของ `main.go:590` ระบบแจกคะแนนโดยยกเว้นเพียงแค่ `InsiderID` และ `JudgeID` ส่งผลให้คนทายถูก (`CorrectGuesserID`) ยังคงได้คะแนน +1 ร่วมกับ Commons อื่น ๆ ตอนจับกุม Insider สำเร็จ
* **ข้อเสนอแนะ:** เพิ่มเงื่อนไขเพื่อข้ามผู้เล่นที่เป็นผู้เดาถูกในลูปการแจกคะแนนให้ตรงตามแผนงานที่ระบุไว้:
  ```go
  if p.ID == room.InsiderID || p.ID == room.JudgeID || p.ID == room.CorrectGuesserID {
      continue
  }
  ```

### 🪑 3.5 โมดูลเสมือนจริงที่ถูกละทิ้ง (Unused PlayerTable Component)
* **ปัญหา:** พบคอมโพเนนต์ `PlayerTable.js` ที่แสดงผลจำลองโต๊ะนั่งบอร์ดเกมทรงสี่เหลี่ยม/วงกลมล้อมโต๊ะอย่างสวยงาม แต่ไม่ได้ถูกเรียกใช้งาน (Import) หรือแสดงผลในหน้าเว็บหลัก (`InsiderGamePage.js`) เลย โดยระบบใช้เพียงรายการคอลัมน์แนวตั้ง 2 ข้างขนาบแทน
* **ข้อเสนอแนะ:** ควรดึงคอมโพเนนต์ `PlayerTable.js` กลับมาเรนเดอร์ไว้ในคอนเทนต์ส่วนกลาง เพื่อจำลองความสมจริงของบอร์ดเกมรอบโต๊ะ และช่วยให้ประหยัดพื้นที่บนหน้าจอมือถือได้ดีขึ้น
